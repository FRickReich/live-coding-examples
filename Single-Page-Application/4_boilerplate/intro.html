Single Page Application

ZU allererst ist es wichtig zu wissen, das wir alle bereits regelmäßige nutzer von Single Page Applications, oder kurz SPA sind. Einige beispiele für Single Page Applications wären GMail, Google Maps, AirBNB, Netflix oder PayPal. All diese anbieter verwenden SPA's damit ihr service einfach zu erweitern, und zu managen ist.

Was ist eine Single Page Application

Single Page Applications sind einzelne seiten, die viele informationen liefern können, und spezifische inhalte aktualisieren, ohne die seite wechseln zu müssen. ---> GMAIL ÖFFNEN http://mail.google.com Wenn wir beispielsweise auf GMail sind, und uns durch unsere emails klicken, sehen wir das sich ausser des hauptinhaltes nichts ändert, die seite läd nicht neu, nur spezifische inhalte werden verändert.

SPA Architektur

Die Single Page Application ist eine Webanwendung oder Webseite, die mit dem benutzer interagiert, indem sie die aktuelle seite dynamisch umschreibt, anstatt neue seiten vom Server zu laden. Dieser Ansatz minimiert die unterbrechung der benutzererfahrung zwischen den aufeinanderfolgendne seiten, so das die anwendung sich mehr wie eine desktopanwendung verhält. Auf den meisten webseiten gibt es viele sich wiederholende Inhalte, Einige davon bleiben gleich, egal wohin der Benutzer geht, zum beispiel der Header, der Footer, die Navigationsleiste, und so weiter. Allgemein bleiben viele elemente des layouts bleich. Single Page Applications machen sich diese wiederholung zu nutze. Bei traditionellen Multi Page Applications, würden wir bei jeder seite die wir anfragen immer die ganze seite, mit allen inhalten geliefert kriegen: --> https://giphy.com/gifs/8F9sWWRDqbcpKoqoz3/fullscreen Bei einer SPA hingegen, bekommt unser browser eine bauanleitung, wie er die seite zusammensetzt, und wo sich was wiederholt oder benutzt wird und fügt dann alles zusammen. --> https://giphy.com/gifs/52FwconEHyJH73EuFt/fullscreen Die geschwindigkeit der SPA kommt dadurch zu stande, das ab diesem punkt nurnoch einzelne informationen ausgetauscht werden müssen, aber keine webseite mehr komplett neu heruntergeladen werden muss, während bei der klassischen seite jetzt immernoch bei jeder anfrage die seite neu geladen werden müsste. --> https://giphy.com/gifs/Xpj7yINzSulzLa8Oq7/fullscreen Da bei Single-Page-Applications die aufgabe des zusammensetzen der webseite auf seiten des browsers liegt, kann der inhalt dynamisch neu geschrieben werden, anstelle komplett neu geladen zu werden. --> https://giphy.com/gifs/KWQVAfIlg5LXQm87cg/fullscreen Anstelle das wir die seite immer neu laden, sehen wir also nur die neuen, ersetzten inhalte. --> https://www.bloomreach.com/binaries/ninecolumn/content/gallery/bloomreach.com_2/blog-images/2018/07/spa/traditional-vs-spa.png Gerade im JavaScript bereich sind SPA's beliebt, weil man schnell und einfach komplette webseiten, oder web-applikationen erschaffen kann, und diese wie desktop apps nutzen kann. Viele Apps auf eurem smartphone sind zum beispiel eigentlich SPAs, die einfach als applikation gestartet werden.

Das konzept MVC

--> https://miro.medium.com/max/875/0*A2NPgocJSjAa2YNe.png MVC steht für Model-View-Controller, eine spezielle struktur für web-applikationen, in der die Applikation in 3 komponenten eingeteilt wird, die voneinander getrennt entwickelt werden können: - Model: Die datenbankanbindung, die mit dem Controller verbunden ist, um daten auszutauschen. - View: Die darstellung der daten auf der HTML seite, die über den controller zum benutzer kommen. - Controller: Die Steuerung der Applikation, hier sitzt die programmlogik, die die daten und die sichtbare webseite verbindet. Der vorteil dieser flexiblen art der programmierung liegen in der einfacheren wiederverwendbarkeit, und wartung, im team ist es auch einfacher an MVC zu arbeiten, wenn jeder seite bestimmten aufgabenbereiche hat. Allerdings ist das setup, um eine MVC aufzubauen komplizierter und man ist möglicherweise abhängig von modulen anderer entwickler.

Was ist React?

React ist ein Javascript framework von Facebook, genau wie Bootstrap von twitter kommt, hat facebook also mit React einen großen einfluss auf das gehabt, was wir als modernes web kennen. React wurde entwickelt um interaktive benutzeroberflächen zu erstellen. Der hauptvorteil von react iast, das man sehr einfach verschiedene Views, oder einzelne kleine teile, genannt komponenten, erstellen kann, die auf anfrage einfach geupdated werden können. Jeder komponent ist in der lage, seine eigenen daten und strukturen zu managen, weswegen man sie fantastisch zu komplexen benutzeroberflächen zusammenfügen kann. Da die applikation dann aus einzelnen komponenten besteht, kann man sie vorneinander getrennt updaten oder warten, ohne die besuchte stelle auf tausenden zeilen html oder javascript suchen zu müssen, und sich gegenebenenfalles mit dem kompletten seitencode ausseiandersetzen zu müssen. React hat eine sogenannte state und prop verwaltung, das sind 2 wichtige konzepte von React. - Props sind eigentdatschaften die von aussen an komponenten übergeben werden um die funktionalität zu erweiten, diese properties können innerhalb des komponenten verwendet werden. Das ist vergleichbar mit parametern, die wir an funktionen übergeben. - States sind eigentschaften die innerhalb einer komponente sind, und einfluss auf den zustand des views haben können. Diese states müssen auf spezielle art und weise gesetzt, genutzt und verändert werden, damit sie in echtzeit genutzt werden können. Dies ist auch der grund, wieso React, React heisst, denn wenn sich ein state innerhalb einer komponente verändert, wird automatisch die renderfunktion aufgerufen, was dazu führt das die neuen daten direkt angezeigt werden, es wird also auf einen state Reagiert, Reacted. React arbeitet ausserdem mit sogenannte hooks, die die states, die verschiedenen lebensphasen eines komponentes, wie zum beispiel den zeitpunkt des ladens des komponenten direkt managen können.

Fangen wir an mit react

Die einfachste möglichkeit am anfang mit react zu arbeiten, ist das terminal program create-react-app, das uns ein rudimentäres react project erstellt, mit dem wir arbeiten können. create-react-app könenn wir entweder mit npm installieren, oder die live-version von nutzen, indem wir npx nutzen. Wir nutzen NPX, damit wir das auch mal benutzen. Wir erstellen einen ordner, gehen hinein und schreiben npx create-react-app und fügen am ende unseren projektnamen an -> npx create-react-app first-app Wir wir sehen installiert es react und einige andere module... also warten wir mal ab. In der zwischenzeit können wir mal in den browser, und uns ein chrome plugin namens "React Developer Tools" installieren, das wir in zukunft öfter mal benutzen werden um uns unsere komponenten im detail anzuschauen. Wir geben bei google "React Developer Tools" ein, klicken das ergebnis auf der chrome web store an, und installieren es in den browser. create-react-app hat uns jetzt eine komplette applikation erstellt, die wir theoretisch einfach ausführen könnten, mit den befehlen die uns aus der package.json im terminal bereitgestellt wurden, aber schauen wir uns erst einmal an was wir jetzt haben. einige dateien und ordner wurden erstellt. Der node_modules ordner ist riesig, wie erwartet, hier ist alles drin was wir zum starten von react brauchen, wenn wir unsere applikation erweitern, brauchen wir gegenenfalles noch andere module, aber für den anfang dürfte das reichen. ausserdem haben wir die ordner public und src. in public finden wir die haupt html datei, die die app ausführen wird, und alles was sonst noch zu der HTML datei gehört, wie zum beispiel das favicon. im src order finden wir unser react projekt. - App.css beinhaltet unser CSS, - App.js ist unsere eigentliche applikation - App.test.js beinhaltet tests für die erfolgreiche ausführung der applikation - index.css beinhaltet css für den body, und vordefinierte elemente. - index.js ist die datei die unsere app in der html datei aufruft. den rest können wir ignorieren, und uns mal die index.js anschauen. .......... jetzt schreiben wir npm start und sehen den live server von create-react-app laufen. wenn wir jetzt in der App.js änderungen machen, sehen wir diese sofort.
